Debugging

Drivers can be debugged remotely via the control system. Setting up debugging requires the following steps:

  • Enable remote debugging on the control system
  • Create a device connection to the control system in Visual StudioÂ® software

Procedures for each task are provided below.

Enable Remote Debugging in Crestron Toolbox

To enable remote debugging on the control system:

  1. Open the Text Console tool in Crestron Toolbox software.
  2. Click the pencil icon to display the Edit Address dialog box.

  3. Click the TCP radio button.
  4. Enter the control system IP address in the IP Address / Hostname field.
  5. Click OK. A command prompt is shown if the connection is successful.
  6. Issue the ssharpappdebug on command. A success message should be returned.

Configure a Device Connection in Visual Studio

To configure a device connection to the control system in Visual Studio:

  1. Open Visual Studio software.
  2. In the Device Toolbar (made visible by selecting Views > Toolbars > Device), click the Device Options button . The Options dialog box is displayed.
  3. Select Device Tools > Devices from the navigation tree.

  4. Configure the following settings in the right-hand pane:
    • Select Windows CE from the Show devices for platform: drop-down menu.
    • Select the control system from the Devices: list.
    • Click Save As to save the control system.
  5. Edit the device properties as follows:
    • Select the device in the Devices: menu and click Properties. A dialog box for editing device properties is displayed.

    • Select Root Folder from the Default output location on device: drop-down menu.
    • Select TCP Connect Transport from the Transport: drop-down menu if not already selected.
    • Fill the Detect when device is disconnected check box if not already filled.
    • Click Configure to display the Configure TCP/IP Transport dialog box, select the Use specific IP address: radio button, and enter control system IP address in the text field. No other settings need to be configured in this dialog.

  6. Click OK to return to the Options dialog box.
  7. Use the Default device: drop-down menu to select the control system as your default device.

Debugging in Visual Studio

To debug in Visual Studio:

  1. Open Visual Studio software.
  2. Select Debug and then Attach to Process.
  3. Select Smart Device from the Transport drop-down menu.
  4. Select Browse in the Qualifier section, and select the control system defined in the procedure above.
  5. Attach to the splusmanagerapp.exe process.
