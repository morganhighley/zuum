Communication

The Communication property contains the following nodes:

Authentication (AuthenticationNode)

The Authentication node specifies all authentication settings for a driver.

Authentication Properties

Property Type Use
UsernameRequired Boolean

Specifies whether a username is required if authentication is possible.
PasswordRequired Boolean Specifies whether a password is required if authentication is possible.
Type String Specifies the type of authentication required. Possible values are "None" and "UsernamePassword".
Required Boolean Specifies if authentication is required to communicate with the device. If this is false, then authentication may still be supported by the device.

SupportsUsername and SupportsPassword defined in DeviceSupport will be used to determine if any authentication is possible with the device.
UsernameMask String No longer used.
PasswordMask String No longer used.
DefaultUsername String The default username that will be used by the driver.
DefaultPassword String The default password that will be used by the driver.

Baud (eComBaudRates)

The Baud node specifies the baud rate of the driver.

If this is an adjustable value on the device, then ComspecAdjustableBaud must be added to UserAdjustableProperties and IsUserAdjustable must be set to "true".

DataBits (eComDataBits)

The DataBits node specifies the data bits configuration for COM device drivers.

If this is an adjustable value on the device, then ComspecAdjustableDataBits must be added to UserAdjustableProperties and IsUserAdjustable must be set to "true".

DataBits Properties

JSON Value Actual Value
0 Not specified
7 7 data bits
8 8 data bits

DeviceId (int)

The DeviceId node specifies the default device ID used by the driver. This value must be set to "0" if a device ID is not used.

If this is an adjustable value on the device, then ComspecAdjustableDeviceId or EthernetAdjustableDeviceId must be added to UserAdjustableProperties and IsUserAdjustable must be set to true.

EnableAutoPolling (bool)

The EnableAutoPolling node specifies whether the driver will start polling the network when it is instantiated. This value should be set to false is the driver does not have a polling sequence or if polling should be disabled until a certain condition is met.

For example, devices that require authentication must disable polling to send the username and password; otherwise, a polling command could be treated by the device as the username and password.

EnableAutoReconnect (bool)

The EnableAutoReconnect node specifies whether the driver will attempt to reconnect to a device if the connection is lost. This value may only be set to true for IP transports as a best practice.

HwHandshake (eComHardwareHandshakeType)

The HwHandshake node specifies the hardware handshake configuration on a COM driver.

If this is an adjustable value on the device, then ComspecAdjustableHardwareHandshaking must be added to UserAdjustableProperties and IsUserAdjustable must be set to "true".

IsSecure (bool)

The IsSecure node specifies whether the driver supports authentication.

IsUserAdjustable (bool)

The IsUserAdjustable node specifies whether the COM port configuration settings, device ID, or Ethernet port may be adjusted by the user. The UserAdjustableProperties properties are used to specify the settings that can be adjusted by the user.

Parity (eComParityType)

The Parity node specifies the parity configuration on a COM driver.

If this is an adjustable value on the device, then ComspecAdjustableParity must be added to UserAdjustableProperties and IsUserAdjustable must be set to "true".

Port (int)

The Port node specifies the default port that an IP driver should use. The driver will not use this port automatically, but applications are able to see the port.

If this is an adjustable value on the device, then EthernetAdjustablePort must be added to UserAdjustableProperties and IsUserAdjustable must be set to "true".

ResponseTimeout (uint)

The ResponseTimeout node specifies the duration (in milliseconds) that must elapse before a polling command is considered to have timed out. Once a command times out, it will be resent when the next polling cycle resets.

TimeBetweenCommands (uint)

The TimeBetweenCommand node specifies the duration (in milliseconds) that driver must wait before sending commands to the device. The minimum value is 250ms. The minimum value is used if a value is specified that is lower.

UserAdjustableProperties (List<string>)

The UserAdjustableProperties node specifies the application settings that may be adjusted by the user. This list includes all COM configuration and IP configuration settings. The values must match the name of the associated property.

Valid values are defined by Crestron.RAD.Common.Enums.eTransportAdjustableProperties. For example, if stop bits were changeable on the device, then the driver would list ComspecAdjustableStopBits as an adjustable property.

WaitForResponse (bool)

The WaitForResponse node specifies whether the driver needs to wait until a valid response is received before sending the next command. This value should only be set to true if responses are ambiguous and the driver cannot discern the context of the response.
