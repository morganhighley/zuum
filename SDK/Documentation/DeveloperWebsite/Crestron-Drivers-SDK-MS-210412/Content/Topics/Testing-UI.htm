User Interface Overview

The following sections describe the functions and features of each screen within the XPanel user interface.

Splash Screen

The splash screen is displayed after starting the program.

The splash screen provides selections for testing various communication methods with your device. Below the RAD logo is a label indicating what type of device you can test, which is based on the program that was loaded to the control system.

In the image above, the label shows “Test Application for Displays," which reflects that the control system was loaded with the “rad_sdk_test_display.lpz” program.

The splash screen provides the following selections:

  • Test IR: Allows for testing of an IR driver
  • Test Serial: Allows for testing of a Serial driver
  • Test Ethernet: Allows for testing of an Ethernet driver
  • Test CEC: Allows for testing of a CEC driver

    • NOTE: A DMPS controller is required to test CEC drivers.

Main Screen

The main screen of the user interface is shown after selecting a connection type from the splash screen. The configuration menu is selected by default.

Each section of the main screen is described below.

Header

The header on the main screen provides the following functions:

  • A drop-down menu for device configuration and SDK interface testing
  • A label describing the selected communication type
  • An indication that the driver is loaded
  • An indication that the control system is connected

Drop-Down Menu

Click the upper-left corner arrow opens a drop-down menu. Clicking the arrow again closes the menu.

The Configuration menu provides a dialog to specify a device driver to load and also any communication properties required for the control system to communicate with the device.

The rest of the menu items relate to the SDK Interfaces that are supported by the device type you are testing. Only interfaces that are actually supported by the device type will show.

The following interfaces are currently supported by the SDK:

  • i-Basic Information
  • i-Connection
  • i-Power
  • i-Inputs
  • i-Volume
  • i-Lamp Hours
  • i-Video Mute
  • i-Navigation
  • i-Keypad
  • i-Media Transport
  • i-Channel
  • i-DVR
  • i-Switching
  • i-Camera Control
  • i-Preset
  • i-Logger
  • i-Custom Command
  • i-Bluray Player
  • i-TV Tuner

Refer to Supported Interfaces for more information on each interface.

Configuration Menu

Select Configuration from the drop-down menu to display settings for configuring device drivers. The Configuration menu provides a dialog to specify a device driver to load and any communication properties required for the control system to communicate with the device.

The communication properties available are dependent on the communication control type that was selected on the splash screen.

Ethernet Control

If Test Ethernet was selected on the splash screen, the Configuration menu provides settings for configuring Ethernet device drivers.

The following settings can configured by editing the appropriate text fields:

  • Driver Filename: The filename of the driver
  • Connection Hostname/IP Address: The IP address of the Ethernet connection
  • Connection Port: The port used by the Ethernet connection
  • Device ID: The IP ID of the device to be tested

Click Save next to a setting to save that setting. Tap Load Driver to load the driver for testing based on the configured settings.

Serial Control

If Test Serial was selected on the splash screen, the Configuration menu provides settings for configuring Serial device drivers.

The following settings can be configured by editing the appropriate text fields:

NOTE: The Serial COM Settings field shows the Serial connection details and is not configurable.

  • Driver Filename: The filename of the driver
  • Device ID: The IP ID of the device to be tested

Click Save next to a setting to save that setting. Tap Load Driver to load the driver for testing based on the configured settings.

IR Control

If Test IR was selected on the splash screen, the Configuration menu provides settings for configuring IR device drivers.

The following settings can be configured by editing the appropriate text fields:

  • Driver Filename: The filename of the driver

Click Save next to a setting to save that setting. Tap Load Driver to load the driver for testing based on the configured settings.
