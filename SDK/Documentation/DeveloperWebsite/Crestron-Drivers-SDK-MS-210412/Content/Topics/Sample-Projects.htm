Sample Projects

The SDK contains sample driver projects that can be used as a reference when creating a new driver. Sample projects have been included for each available device type and transport method. For example, the AvReceiver sample project has both an IP (SIMPL#) and Serial (SIMPL#Pro) example.

Except for the ExtensionDevice samples, which are dedicated example projects, these sample projects are real drivers that have been tested with their respective devices.

NOTE: All included sample projects target both 3-Series® and 4-Series™ control processors, so Visual Studio 2008 with the SIMPL# plugin is required to build them.

Extract the Sample Projects

The sample projects are contained within /SDK/Samples/Samples.zip. Extract the entire directory to a single location on your workstation to ensure that the relative paths in the projects remain correct.

Samples.sln at the top level of the directory is the solution that contains each of the driver sample projects.

Drivers Directory

The Drivers directory contains the sample project source code, grouped by device type, manufacturer, and driver name.

Packages Directory

The packages directory contains the SDK libraries that these sample projects reference. These are the same libraries that are distributed elsewhere within the SDK, but they are repeated here to make it easy to compile the samples.

Build the Samples

Prior to building, ensure the development setup steps have been followed as specified in Installation and Setup. In particular, Visual Studio 2008 SP1, Crestron Database v200 or newer, and the SIMPL# Plugin need to be installed.

To build the samples:

  1. Extract Samples.zip to a directory of your choosing. Extract all the files and do not move them relative to each other to avoid breaking the relative paths in the project files.
  2. Open Samples.sln in Visual Studio 2008.
  3. Build the solution.
  4. To create a .pkg file from the compiled DLL, follow the instructions located in /SDK/ManifestUtil/README.txt.

    5. The README file also contains instructions on how to set up Visual Studio macros to automatically create .pkg files when the driver builds successfully

NOTE: Some sample builds of drivers have the SdkVersion set to 99.x.x.x to match the SDK version, but some control systems will check this version number and refuse to load the driver because the major version indicates it may be incompatible. Before building and loading a sample, ensure the SdkVersion major version is not 99. If it is, change it to 1.00.000.

Use the Samples to Create a New Driver

Many devices use similar communication protocols and command schemes, so it is sometimes easier to start with a similar driver and modify it to fit then new device than to start completely from scratch. However, there are some elements that must be unique to each driver.

To use an existing sample driver as the baseline for a new driver:

  1. Copy the project directory and rename the files to match the name of the new driver.
  2. In each of the files (including the .cs, .csproj, and .json files), replace occurrences of the old driver name with the new one. Ensure the paths in the .csproj files match the newly renamed files.
  3. Create a new GUID and replace the <ProjectGuid> in the .csproj file with your new one.

    4. One way to create a new GUID is to use the generator built into Visual Studio under Tools > Create GUID. This GUID must still be copied into the .csproj file, however.

  4. Create another new GUID and replace the “Guid” in the driver JSON file with this one.
  5. Update the device information in the JSON file, including the manufacturer, supported models, and so forth.
  6. Ensure that the “SdkVersion” field is set appropriately. Refer to the note in Build the Samples above for more information.
  7. Set the version and other assembly information in Properties/AssemblyInfo.cs.
  8. Modify the C# sources and .json file as necessary for your driver.
