The IDevicePropertyT generic type exposes the following members.

Attributes (Inherited from IDeviceProperty.)
AvailableValues
Available values for the property; The property’s value can only be set to one of these.

In addition to being the property’s value, each ID also refers to a localized string that is display name of this value. The localized string can be retrieved via the ILocalizedDevice capability.

Also, based on the RenderHint, this may also contain labels for each value defined by MinValue through MaxValue. For example, if the value range is 0 to 3, this may define the labels for those values as "Off", "Slow", "Medium" and "Fast", respectively.

MinValue
Minimum value allowed for numeric values; For strings, minimum length. Not applicable for all other non-numeric values.
ParentPropertyKey
Used to associate this property with another property (in addition to any RenderHint values that imply relationships).

For example, a device may define 2 properties, where the 1st property is a Boolean on-off switch, and the 2nd property has a render hint of Status which indicates the status of the device turning on. The 2nd property will refer the 1st property key here to indicate that it is associated with the 1st property.

Another example would be if a device had 2 sets of setpoints (i.e. 2 properties with SetpointTarget and another 2 properties SetpointActual). This key would indicate which target setpoint belongs with which actual setpoint (or vice-versa).

This may be null if the device only contains one set of associated properties. Using the above example, if a device only has one target and one actual setpoint property, the relationship is already implied by the render hint.

State

Changes will be notified via the IDeviceWithProperties capability's event PropertyStateChanged.

(Inherited from IDeviceProperty.)
StepSize
Amount to increment or decrement the value at a time; Applicable only when the Type refers to a numeric value.

If the type is Float or Double, this also defines the precision of the floating point value.

For example, if this value is 0.2, it would mean valid values could be 0, 0.2, 0.4, etc... Any invalid values passed to the SetValueT(String, T) method will be adjusted to the nearest valid value in the direction the value is being adjusted.
