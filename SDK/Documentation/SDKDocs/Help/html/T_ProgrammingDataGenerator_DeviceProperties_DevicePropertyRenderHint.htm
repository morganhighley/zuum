DevicePropertyRenderHint Enumeration

[This is preliminary documentation and is subject to change.]

Defines hints for user interfaces as to the intended usage of a device property; Specified by the interface property RenderHint.

public enum DevicePropertyRenderHint
Members
  Member nameValueDescription
None0 Indicates the property value should be displayed using normal editing controls, such as a check box for booleans, a text box for string, etc...
OnOff1 Indicates the property is intended to be displayed as on/off switch.

Only valid for properties whose type is Boolean.

OnOffValues2 Indicates the property is an additional value that can be set, in addition to an associated property whose render hint is OnOff.

In other words, there will be 2 properties. One will be the property that turns on/off the device feature. The other will be the one that configures that device feature.

For example, turning on a strobe light may offer a preset list of flashing patterns to choose from. The intention here is that a user interface might want to show the 2 properties closely coupled together. This property’s AvailableValues will be populated.

SetpointTarget3 Indicates the property is intended to be displayed as range of values the user can target, for example, as a slider or a numeric spinner.
SetpointActual4 Indicates the property is intended to be displayed as the actual/current value when there is another property with the hint of SetpointTarget.
Slider5 Indicates the property is intended to be displayed as slider or similar control.

How exactly the slider is rendered can vary depending on the number of values specified by the range MinValue and MaxValue and the StepSize between them.

The Units should also be factored in when rendering the slider.

If the AvailableValues is populated, it indicates custom labels from the device for each value (e.g. Slow, Medium, Fast).

Status6 Indicates the property is the status of an associated property.

For example, turning on a pump may require a "priming" phase. One property would exist to turn the pump on/off. In order to define the "priming" status of the 1st property, a 2nd property would exist with this render hint. Finally, the 2nd property would have its ParentPropertyKey set to the Key of the 1st property.

The intention here is that a user interface might want to show the 2 properties closely coupled together since one is the status of the other.
