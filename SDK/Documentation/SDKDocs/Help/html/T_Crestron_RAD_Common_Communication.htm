Communication Class

Specifies any communication settings on a driver such as the COM port specifications, polling settings, and command intervals.

Authentication
Specifies any authentication settings for communciation with the device.

This is referring to APIs that require the driver to send a username and/or password.

Baud
A portion of the COM port specifications.

If this driver does not use the COM port, this should be set to NotSpecfied.

CommunicationType
The type of communication used to communicate with the third-party device.
DataBits
A portion of the COM port specifications.

If this driver does not use the COM port, this should be set to NotSpecfied.

DeviceId
Specifies the default device ID that should be used for communication.

This is used when the API requires an ID to be included in commands sent to the device.

EnableAutoPolling
Specifies if the driver should poll automatically.

If this is set to false, then the driver will not poll unless the driver sets PollingEnabled to true.

This should be set to true even if SupportsUnsolicitedFeedback is set to true. This allows the driver to poll at least once to get initial states.

EnableAutoReconnect
Specifies if the driver should reconnect to a device automatically if there was a disconnect.

This is not used if the driver uses the COM port.

HwHandshake
A portion of the COM port specifications.

If this driver does not use the COM port, this should be set to NotSpecfied.

IpProtocol
Specifies the protocol used for ethernet communication.

This should be set to NotSpecified if the driver doesn't use ethernet for communication.

IsSecure
Not used by the framework.
IsUserAdjustable
Specifies if there are any values in the Communciation node that can be modified.

This is used by applications to potentially allow users to change the baud rate and other settings. Applications will look at UserAdjustableProperties to find which properties can be changed.

Parity
A portion of the COM port specifications.

If this driver does not use the COM port, this should be set to NotSpecfied.

Port
The ethernet port used for communication

If this driver is not using ethernet for communication, this should be set to 0.

Protocol
The protocol used for communication.

If this driver does not use the COM port, this should be set to NotSpecfied.

ResponseTimeout
Specifies the amount of time in ms to wait before considering a polling command to be timed out.
StopBits
A portion of the COM port specifications.

If this driver does not use the COM port, this should be set to NotSpecfied.

SwHandshake
A portion of the COM port specifications.

If this driver does not use the COM port, this should be set to NotSpecfied.

TimeBetweenCommands
Specifies the amount of time in ms to wait in between sending commands.

This value must be greater than 0 and a multiple of 250. If it is not, it will be automatically quantized to be a multiple of 250. Example: A value of 200 will be treated as 250 internally and a a value of 251 will be treated as 500 internally.

If the driver wishes to specify a value less than 250, then it should set TimeBetweenCommandsOverride.

TimeBetweenCommandsOverride
Specifies the amount of time in ms to wait in between sending commands.

If this is not null, this will be used instead of TimeBetweenCommands.

This value may be 0 and must be a multiple of 25.

This value should only be set in the driver's embedded JSON file and should not be referenced by the driver.

UserAdjustableProperties
Specifies which properties are adjustable. This is used by applications.

Valid values are defined in eTransportAdjustableProperties.

WaitForResponse
Specifies if the driver should wait for a response to a previous command before sending another command. This is tracked using ValidateResponse when it returns ValidatedRxData with Ready set to true.
