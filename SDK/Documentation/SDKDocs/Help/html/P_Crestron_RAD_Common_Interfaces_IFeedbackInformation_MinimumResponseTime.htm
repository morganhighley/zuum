IFeedbackInformationMinimumResponseTime Property

[This is preliminary documentation and is subject to change.]

Approximate minimum number of milliseconds the device or driver will report state/feedback changes.

The formula for this value is typically: [Polling Interval (if any)] + [Device Response Time] + [Response Processing Time].

This may be 0 or a very small value if the device responds quickly and driver processing time is negligible.

Amongst other reasons, this property can be used to write smarter code that waits at least this amount of time before processing changes to provide optimize updates to higher level components.

If no capabilities of the device support feedback, this can be ignored.
