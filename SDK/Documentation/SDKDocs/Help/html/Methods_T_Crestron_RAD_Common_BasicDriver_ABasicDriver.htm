OverridePassword
Allows applications to set the password the driver should use when authenticating with the device. This can be set before or after initializing and connecting the driver. If a PasswordKey is supplied, then this value will not be used unless there is no password in CrestronDataStore.

Driver developers - this must be implemented in the driver if the driver supports authentication with the device.

OverrideUsername
Allows applications to set the username the driver should use when authenticating with the device. This can be set before or after initializing and connecting the driver. If a UsernameKey is supplied, then this value will not be used unless there is no username in CrestronDataStore.

Driver developers - this must be implemented in the driver if the driver supports authentication with the device.

RegisterWithCloud
This must be invoked if the application desires the driver to report to the cloud. The driver will report any applicable feedback changes to the cloud, and it will allow the cloud to control the driver.
RemoveCustomCommandByName

RemoveUserAttribute
Only attributes that have been added using AddUserAttribute(UserAttributeType, String, String, String, Boolean, UserAttributeRequiredForConnectionType) can be removed. User Attributes defined in the driver's json file cannot be removed.
RetrieveUserAttributes
Provides a list of available user attributes.

This collection is defined by the following JSON driver data class: DriverData.CrestronSerialDeviceApi.UserAttributes, plus any user attributes that have been added dynamically by the driver using AddUserAttribute(UserAttributeType, String, String, String, Boolean, UserAttributeRequiredForConnectionType).

SaveSetting
Save a setting for this device by key which will be persisted through reboots and power loss.

This can be useful for devices that don't remember their own settings or can be used to return a device to its previous state after a power failure/reboot/etc.

To retreive the value of the setting use GetSetting(String).
