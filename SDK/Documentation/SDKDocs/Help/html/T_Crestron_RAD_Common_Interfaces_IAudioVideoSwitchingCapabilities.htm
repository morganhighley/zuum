Properties
  NameDescription
AutoDetectConfiguration
Specifies if this driver is able to determine what the available routable inputs and outputs are.
CompatibleInputs
Returns a list of input extenders that could be used on this system

This is intended to be used in conjuction with SetInputExtender(UInt32, IAudioVideoExtender).

CompatibleOutputs
Returns a list of output extenders that could be used on this system. Every extender will list its input/output connectors as well as any inputs/outputs that are routable on it exclusivly.

This is intended to be used in conjuction with SetOutputExtender(UInt32, IAudioVideoExtender).

SupportsAudioBreakaway
Specifies if this switching device supports routing audio and video from different input extenders to one extender output.

Methods
  NameDescription
GetRoutableInputs
Returns the current detected/set input extenders on the device that can be routed to the outputs returned by GetRoutableInputs using RouteVideoInput(String, String) and/or RouteAudioInput(String, String). If the device does not support audio-breakaway, then both of those methods will perform the same function and throw their respective events.
GetRoutableOutputs
Returns the currently detected/set output extenders on the device that inputs returned by GetRoutableInputs can be routed to using RouteVideoInput(String, String) and/or RouteAudioInput(String, String). If the device does not support audio-breakaway, then both of those methods will perform the same function and throw their respective events.

Returns null if there is no option to route inputs to outputs

RouteAudioInput
Routes audio from the specified input to the specified output. This will route both audio and video if audio-breakaway is not support on the device.

To clear the route, specify null for the input.

RouteVideoInput
Routes video from the specified input to the specified output. This will route both audio and video if audio-breakaway is not support on the device.

To clear the route, specify null for the input.

SetInputExtender
This can be used to switch out defined inputs with a compatiable ones.
SetOutputExtender
This can be used to switch out defined outputs with a compatiable ones. For example, if a 3x1 HDMI switcher was going to be connected via HDMI1 to Output3, then the application should call this method updating the driver stating that HDMI1 is now a different extender.

If the driver can't detect HDBaseT extenders, then the application would use this to tell the driver which compatible extender is being used on the output.

Events

RoutableInputsChanged
Raised when the list of routable inputs has changed.

Presumably, subscribers of this event will have called GetRoutableInputs method once to retrieve a copy of the list of members. Based on the arguments of this event, the subscribers can then update their copy of the list without asking this device for the full list again (unless the event was a Reset; In that case, the full list would need to be retrieved again anyway).

RoutableOutputsChanged
Raised when the list of routable outputs has changed.

Based on the arguments of this event, the subscribers can then update their copy of the list without asking this device for the full list again (unless the event was a Reset; In that case, the full list would need to be retrieved again anyway).
