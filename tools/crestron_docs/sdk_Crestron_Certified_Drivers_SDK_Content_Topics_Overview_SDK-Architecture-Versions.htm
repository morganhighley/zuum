Overview / SDK Architecture Versions

SDK Architecture Versions

As of Crestron Drivers SDK version 21.x, a V2 architecture version (Entity Model) is now available for creating drivers. In contrast to the V1 SDK architecture (RAD Framework), which defines and maintain C# interfaces to create and expand capabilities for device types, the Entity Model architecture defines data structures that describe the device's capabilities in terms of properties, commands, and events.

The Entity Model architecture differs from the RAD Framework architecture in the following ways:

  • The only required assembly references are Crestron.DeviceDrivers.EntityModel and Crestron.DeviceDrivers.SDK.
  • The format of the Driver JSON file has changed.
  • User Attributes, transport-specific Initialize methods, and IDriverInitialization have been replaced with a single unified configuration mechanism.
  • Capability interfaces have been replaced by a capability's properties, commands, events, and types.
  • A device type and its capabilities are now defined in data and not in specific C# interfaces.
  • All interaction between the application and the driver has been funneled through a single DriverController class using immutable data structures.
  • The Driver SDK helps facilitate the routing of these calls to specific object instances when needed.

Certain device types can only be created using a specific version of the SDK architecture, while other types can be created using either architecture. For more information, refer to Supported Device Types

Instructions for creating drivers using these architectures, including API references and best practices, are provided in the following sections:

  • Driver SDK V1 (RAD Framework)
  • Driver SDK V2 (Entity Model)
