Entry

entryUserManagement

This capability provides support for interacting with a user management system associated with a entry device (lock, door, garage door, gate, etc.).

Due to the security nature of this device, any command that fails should indicate that in the command result and optionally include a user friendly error message indicating the reason for the failure.

For consumers of this class, it is important to alert the end user of any failed commands when possible (if not, a user may attempt to issue the remove command, and believe it succeeded when in fact it failed.)

WeeklyAccessTime

Fully Qualified Id: entryUserManagement:WeeklyAccessTime

Defines access for a weekly time period.

Properties
Name Details
activeDays

Defines which days of the week access is being allowed.

Type: ICollection<DayOfWeek>
startTimeInMinutes

Defines the time in minutes of when access will begin for each day defined in "activeDays".

The time is expected to be in local time.

Example: A start time of 8:30 am would be 510 ((8*60)+30).

Note: 0 indicates 12:00 am.

For all-day access, set "startTimeInMinutes" to 0 and "stopTimeInMinutes" to 1440.

Type: long
stopTimeInMinutes

Defines the time in minutes of when access will end for each day defined in "activeDays".

The time is expected to be in local time.

Example: A stop time of 5:00 pm would be 1020 (17*60).

Note: 1440 indicates 12:00 am the following day.

For all-day access, set "startTimeInMinutes" to 0 and "stopTimeInMinutes" to 1440.

Type: long

This command returns an AddUserResult which contains the user id of the newly created user and an ErrorCode indicating the result of the operation.

This capability provides control for the core features of a lock.

Types

State

Fully Qualified Id: lock:State

Lists the various states that a lock can have.

The values in this enum are specific to the lock capability.

A lock may have other states, but those would be captured by other capabilities (i.e. low battery, offline, etc.).

Note: This enum should not be used to describe the state of doors, gates, garage doors, and other entryways. For doors, gates, garage doors, or other entryways, see the "entry" capability.

EventType

Fully Qualified Id: lock:EventType

Lists the various events that can occur on a lock.

The values in this enum are specific to the lock capability.

A lock may have other events that occur, but those would be captured by other capabilities (i.e. user management, create/modify scheduled events, etc.).

When selecting a value from this enum, choose the value that is most specific. For example, if the the door was locked from the keypad, use "LockedFromKeypad" instead of "Locked".

Definition
Copy

Fully Qualified Id: lock:setAutoLock

Sets the state of the auto lock feature.

The auto lock feature will automatically lock the door after a certain delay.

deviceName

Capability to be implemented by a device or device component which supports a configurable name

This capability is reserved for devices which report their own user-configurable name, and potentially the ability to set that name via the API.

This capability should not be "faked" where the driver emulates this feature.

This "name" is for describing and identifying the device in some user interface. Sometimes the device supports configuring its host name on the network, but that is not the intent of this interface.

This is NOT the label associated with a device component. For that, see the deviceComponentLabel capability.

For example, a power controller may have multiple banks, labeled something like "Bank 2" or "Aux" on the power controller chassis. It may also support a software-defined "name" for the bank that may be changed via the device user interface. This capability would be used to report that software-defined name, not the "Bank 2" or "Aux" label.

readyIndicator

Capability to be implemented by a device which indicates when it is "ready" to accept commands. This is not necessarily the same as an "online" flag. The device may be online, but not "ready" yet.

Properties

isReady

Fully Qualified Id: readyIndicator:isReady

True when the device is ready to receive commands and return current information via any properties or commands with return values, as well as raise (feedback) events (if supported).

When this is false, it typically means the device is either offline or just not ready to communicate yet (i.e. online, but still loading).

Unless otherwise noted in a specific device capability property, command or event, all interactions with the device will not be available or valid until this property turns true.

Methods and properties may still be called while this property is false, however their values will be deemed unreliable (and will be set to either their last value or the default value) until this property changes to true.

If an error can occur that will prevent the device from ever being ready, the errorIndicator capability should be implemented in order to raise the error.

The device may also become ready, but have its errorIndicator:errorCodes property already set. This means the device is functional, but has errors that may prevent it from functioning properly.

All other errors will be raised after the device is ready. If the device stays ready, it means the device is functional, but has errors that may prevent it from functioning properly. If the device changes to not ready, it means the error is causing the device to be non-functional.

    type: boolean
