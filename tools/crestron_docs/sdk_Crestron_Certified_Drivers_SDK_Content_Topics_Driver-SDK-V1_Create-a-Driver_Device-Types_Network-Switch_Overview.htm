Driver SDK V1 (RAD Framework) / Create a Driver / Nonbasic Device Types / Overview

Overview

Drivers for network switches follow a capability-based pattern, where core device functions are divided into different capability interfaces. Your driver can choose to implement these interfaces depending on the functions that your network switch supports. A SupportedFeatures enum is included within each capability interface that allows for a more granular representation of the device capabilities.

Capability interfaces all implement the same IDeviceCapability base interface. These capabilities are implemented by the main driver class which is derived from ABasicDriver. The capability pattern allows the consuming application to query the driver for its capabilities and to adjust the features made available to the user based on the features supported by the device.

Implementation

The network switch driver must implement the following capabilities:

  • INetworkPortContainer
  • IReadyIndicator

The network switch driver may implement the following capabilities depending on the device functions:

  • IPowerBankContainer
  • IComputePlatformDiagnostics
  • IDeviceName
  • IDeviceDiagnostics2
  • INetworkAdapterInformation
  • IRebootController
  • IPowerBankControls

The INetworkPortContainer capability is required for network switch drivers, as this tells the consuming application that the device is a network switch. Network switch drivers also require the IReadyIndicator capability, which tells the consuming application that the network switch is ready to receive commands and provide accurate feedback. All other capabilities are optional and are used to further define the device to the consuming application.

The IPowerBankContainer capability is used if the network switch has ports with Power over Ethernet (PoE) and may only return ports that support PoE. The IPowerBankControls capability may be implemented at the network switch level to enable or disable PoE on all ports that support it.

The INetworkPortContainer capability defines the individual ports that the network switch contains. Network ports are represented by an instance of IDeviceWithCapabilities with the INetworkPortContainerItem capability. If the port also supports PoE, then it will be also be represented by the IPowerBank interface.

NOTE: IPowerBank is not a capability. It derives from the IDeviceWithCapabilities interface, meaning that each network port can implement its own capabilities such as IPowerBankControls.

Network ports must implement IDeviceWithCapabilities and must include an implementation of INetworkPortContainerItem as a capability. The capability may not be implemented directly by the class implementing IDeviceWithCapabilities.

NOTE: Certain consuming applications may not utilize all the capabilities supported by a network switch at the time of creating your driver. Regardless, all supported capabilities should still be implemented by the driver since the consuming application may be updated in the future to support the capability. This allows the driver to be feature ready without requiring an update when a new version of the consuming application is released.

Capabilities

Each network switch capability is described in the sections that follow. For more detailed information about each capability, refer to the C# API documentation for network switches.

NOTE: Capabilities should be implemented only when the network switch hardware supports the feature. Features should not be implemented via software in the driver.

INetworkPortContainer

The INetworkPortContainer capability is the base capability for network switches and must be implemented in all drivers. This capability defines a device that contains one or more network ports.

INetworkAdapterInformation

The INetworkAdapterInformation capability is used to report static information about any network adapters on the network switch device.

IPowerBankControls

The IPowerBankControls capability contains functions needed to control PoE settings on the network port. The capability will control all PoE ports wen the capability exists on the main driver class.

INetworkPortControl

The INetworkPortControl capability is used to enable or disable a specific port on the network switch.

INetworkPortStatus

The INetworkPortStatus capability reports status items for the port, such as the current link status.

Related Capabilities

The following capabilities are not specific to network switches but may be useful when creating the driver. For more information, refer to the C# API documentation for network switches.

  • IReadyIndicator
  • IDeviceComponentLabel
  • IRebootController
  • IDeviceName
  • IDeviceDiagnostics2
  • IComputePlatformDiagnostics
