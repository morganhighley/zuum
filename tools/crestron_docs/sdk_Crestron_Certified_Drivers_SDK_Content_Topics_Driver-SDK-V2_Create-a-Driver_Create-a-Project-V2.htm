Create a Project

The steps required to create a driver project differ depending on whether Visual Studio® 2019, 2022, or 2008 software is being used for your development environment.

Create a Visual Studio 2019 or 2022 Project

  1. Open Visual Studio software.
  2. Create a new .NET Framework Class Library project.

    3. NOTE: In the example above, .NET Framework 4.7.2 is used, but any version after .NET Framework 3.5 can be used up to and including version 4.7.2.

  3. Name the driver based on the manufacturer, model, and communication type.

    4. For example, a driver with the following attributes would be named "Platform_TestManufacturer_TestModel_IP".

    • Manufacturer: TestManufacturer
    • Model: TestModel
    • Communication Type: IP
    • Device Type: Platform
  4. If your driver requires SIMPL# references, add the corresponding Crestron NuGet packages.
    1. Open the NuGet package manager.

    2. Navigate to Browse.
    3. Enter the search term "Crestron."

    4. Install one of the following packages as required by your driver:
      • Crestron.SimplSharp.SDK.Library
      • Crestron.SimplSharp.SDK.ProgramLibrary

  5. Add references to the following NuGet packages:
    • Crestron.DeviceDrivers.DevKit: This package includes references to all relevant SDK assemblies.
    • Crestron.DeviceDrivers.ManifestUtil: This package downloads ManifestUtil and runs it automatically after your driver is built. It will not add a reference to the ManifestUtil assembly from your compiled driver.

    6. NOTES: 

    • References to other Crestron.Drivers NuGet packages are not required. Crestron.Drivers.Program is for driver consumption and not development. Crestron.Drivers.ProgramInterface contains shared dependencies of the other packages and is included with Crestron.DeviceDrivers.DevKit as a dependency.
    • It is recommended to use the PackageReference format when adding NuGet packages, but packages.config is also supported.

Create a Visual Studio 2008 Project

  1. Open Visual Studio software.
  2. Navigate to File > New > Project.
  3. Select Crestron from the Installed > Visual C# selections, then select SIMPL # Library.

  4. Name the driver based on the manufacturer, model, and communication type.

    5. For example, a driver with the following attributes would be named "Platform_TestManufacturer_TestModel_IP".

    • Manufacturer: TestManufacturer
    • Model: TestModel
    • Communication Type: IP
    • Device Type: Platform
  5. Add references to Crestron.DeviceDrivers.EntityModel.dll and Crestron.DeviceDrivers.SDK.dll.

    6. NOTE: References to other SDK assemblies are not required (even for Serial drivers) and should not be added unless otherwise necessary for your own resources (such as reuse of existing code).
