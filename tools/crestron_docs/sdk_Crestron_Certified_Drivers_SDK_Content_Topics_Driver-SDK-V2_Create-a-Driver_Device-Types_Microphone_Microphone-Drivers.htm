Microphone Drivers

Any entity model driver that supports microphone features may implement microphone capabilities to expose them to the application. These capabilities may encompass many types of single microphones and microphone arrays. Microphone arrays may expose additional features for sound sources that are accounted for within microphone capabilities.

Before adding microphone capabilities to a driver, it is important to understand the differences between a microphone device and audio media:

  • The primary function of a microphone device is to capture and act a source for audio signals. The driver implementation will contain audio media components. Devices that use audio in different ways most likely fall under different classifications.
  • Audio media covers a wide range of implementations and is available in any device type that implements the mediaDevice capability. Example use cases for audio media that are not microphone implementations include audio mixers where audio is routed, processed, or computed but is not captured; media players where audio is sourced but is not captured; and external inputs where an external signal is forwarded into the system but not captured (even if coming from a device that would classify as a microphone).

NOTE: An exception to the definitions above is microphone arrays, where multiple microphones are in the same unit and the signals are mixed and processed as one device. This scenario is why audioMixer capabilities are included in microphone devices, as the configuration classifies as a microphone device and an audio mixer simultaneously.

In summary, a microphone device is primarily used for capturing audio, whereas audio media can be any audio signal that can be implemented in any device that has the mediaDevice capability.

To locate a self-test plan for microphone device drivers, refer to Microphone Test Plans.
