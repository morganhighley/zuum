Driver SDK V2 (Entity Model) / SDK Framework / Configuration Item Definitions

Configuration Item Definitions

For the complete API reference for Configuration Item settings, refer to the C# API documentation.

NOTE: The Id property is always arbitrary and needs to be used consistently within your driver. However, any Id properties in the sections below with underscores (such as _Username_) are ones that the SDK uses internally, and it is recommended that your driver use those as well (if applicable). The SDK includes certain transport implementations which can populate ID properties using the standard SDK Ids.

Generic Configuration Items

The following are valid ValueType parameters for configuration items.

  • None: Message box (default when omitted)
  • String: Generic string
  • Number: Number (limited to 0–65535 for compatibility with SIMPL)
  • Boolean: true or false
  • Hex: Hexadecimal number (limited to 0x0000–0xFFFF for compatibility with SIMPL)

The following example shows some examples of different ValueType scenarios:

Copy 
[
    {
        "Id": "MyMessageBox",
        "UsageContext": "Generic.Prompt",
        "Title": "The title of the box",
        "Description": "The longer message"
    },
    {
        "Id": "MyCustomPrompt",
        "UsageContext": "Generic.Prompt",
        "Title": "Some title",
        "Description": "Description of what to enter",
        "ValueType": "String",
        "Required": true
    },
    {
        "Id": "MyUrlItem",
        "UsageContext": "Generic.Url",
        "Title": "Log in here",
        "Description": "Click 'log in now' and enter your credentials",
        "ValueType": "String",
        "CurrentValue": "www.crestron.com",
        "CurrentValueDisplayOverride": "log in now",
        "Persistent": true,
        "ReadOnly": true
    }
]

Invisible Configuration Items (Saved Data/Settings)

When a UsageContext of Data is used, the item is not shown to the user. Instead, it is a saved value that is updated by the driver through the ConfigurationItemsUpdated event. When the driver is loaded again after a restart, the application will pass the item back.

To raise this event in a driver using the DelegateDataDrivenConfigurationController class, call DelegateDataDrivenConfigurationController.NotifyValuesChanged() with the updated values.

While these items are not shown to the user, they should be included in a configuration step to provide the application with the default values.

Refer to the following example:

Copy 
[
    {
        "Id": "MyAuthKey",
        "UsageContext": "Data",
        "Masked": true,
        "ValueType": "String"
    },
    {
        "Id": "SomeOtherSetting",
        "UsageContext": "Data",
        "ValueType": "String"
    }
]

Authentication Configuration Items

Authentication configuration items should be included in the same step as IP configuration items when applicable.

Refer to the following example:

Copy 
[
  {
    "Id": "_Username_",
    "Persistent": true,
    "Required": true,
    "Title": "Username",
    "UsageContext": "Authentication.Username",
    "ValueType": "String"
  },
  {
    "Id": "_Password_",
    "Persistent": true,
    "Required": true,
    "Masked": true,
    "Title": "Password",
    "UsageContext": "Authentication.Password",
    "ValueType": "String"
  }
]

IP Configuration Items

The DefaultValue for the _Port_ ID must be changed to match your device's API.

Refer to the following example:

Copy 
[
  {
    "Id": "_Host_",
    "Persistent": true,
    "Required": true,
    "Title": "IP Address / Hostname",
    "UsageContext": "Network.Address",
    "ValueType": "String"
  },
  {
    "Id": "_Port_",
    "DefaultValue": 80,
    "MaxValue": 65535,
    "MinValue": 1,
    "Persistent": true,
    "Required": true,
    "Title": "Port",
    "UsageContext": "Network.Port",
    "ValueType": "Number"
  }
]

Serial Configuration Items

The CurrentValue and AvailableValues should be changed to match what your device supports. The _ChannelId_ is used with the TransportTx delegate passed to the driver to communicate via the hardware port.

NOTE: The remaining items are read-only, as it is the application's job to configure the port. The driver is only telling it what it believes the valid settings are.

CEC Configuration Items

The _ChannelId_ is used with the TransportTx delegate passed to the driver to communicate via the hardware port.

Refer to the following example:

Copy 
[
  {
    "Id": "_ChannelId_",
    "Persistent": true,
    "Required": true,
    "Title": "CEC Port",
    "UsageContext": "Cec.ChannelId",
    "ValueType": "String"
  }
]
