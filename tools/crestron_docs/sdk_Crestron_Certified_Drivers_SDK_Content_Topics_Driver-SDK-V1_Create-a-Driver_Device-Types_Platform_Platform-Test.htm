Testing and Debugging

The following sections describe information that will help you to test and debug your platform drivers.

Common Issues

The table below describes common issues that can occur with platform drivers.

Problem Solution(s)
The driver PKG file was not generated by the Manifest Utility tool. 1. Verify that your driver’s JSON file is correct. Compare the file to other existing drivers, such as the samples provided. Also use an online JSON validation tool to detect small errors.

2. Verify that you are using the latest version of the Manifest Utility tool provided in the Crestron Drivers SDK package (refer to Resources for download links).
The driver PKG file does not contain all required files. 1. Verify that any platform-dependent libraries are included in the Dependencies section of the JSON data file.

2. Ensure all files listed in Dependencies are copied to the same folder as the main driver DLL file when running the Manifest Utility tool.

3. If the platform driver contains Crestron Home Extension drivers, verify that the IncludeInPkg folder was provided to the Manifest Utility tool.

4. Verify that the UI definition and translation files are in the IncludeInPkg folder within the correct subfolders (UiDefinitions and Translations, respectively).
The driver cannot be added to Crestron Home. 1. Verify that your driver does not throw any exceptions when adding it to the system. Exceptions that have occurred can be found in the error log (console command err sys).

2. Verify that your driver fully implements the AGateway and AGatewayProtocol abstract classes.
The paired devices cannot be added to Crestron Home.

1. Verify that the instances for your paired devices are not null and have not been disposed of.

2. Verify that your paired devices do not throw any exceptions when adding them to the system. Exceptions that have occurred can be found in the error log (console command err sys).
The paired devices do not appear in Crestron Home. 1. Verify that you are notifying Crestron Home that a paired device has been added by calling the AddPairedDevice method correctly in your gateway protocol class.

2. Ensure that your gateway driver is reporting as online within Crestron Home.
The paired devices are showing as missing in Crestron Home 1. After a system restart, it is expected that the paired devices will appear as missing in Crestron Home if your driver has not been initialized yet. If they remain missing, then ensure that your platform driver is instantiating the paired devices and providing them to Crestron Home by calling the AddPairedDevice method.

2. Verify that the instances for your paired devices are not null and have not been disposed of.

Test with Crestron Home

To test and debug the platform driver within Crestron Home, refer to Crestron Home OS Testing.
