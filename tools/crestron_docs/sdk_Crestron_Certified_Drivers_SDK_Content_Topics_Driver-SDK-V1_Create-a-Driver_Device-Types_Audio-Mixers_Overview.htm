Driver SDK V1 (RAD Framework) / Create a Driver / Nonbasic Device Types / Overview

Overview

Drivers for audio mixers follow a capability-based pattern, where core device functions are divided into different capability interfaces. Your driver can choose to implement these interfaces depending on the functions that your power controller supports.

Capability interfaces all implement the same IDeviceCapability base interface. The main capability class, which is derived from ABasicDriver, implements IDeviceWithCapabilities.TryGetCapability<T>(). Applications use this to query for and retrieve implementations of these capability interfaces. Therefore, the main driver class does not need to implement every interface directly and may instead have another object designed to handle a particular feature. The capability pattern allows the consuming application to query the driver for its capabilities and to adjust the features made available to the user based on the features supported by the device.

Implementation

The audio mixer driver must implement the following capabilities:

  • IAudioMixer
  • IAudioChannel

The audio mixer driver may implement the following capabilities depending on the device functions:

  • IAudioChannelGainController
  • IAudioSignalDetection
  • IChildDeviceContainer
  • IDeviceDiagnostics
  • IDeviceIdentification
  • IDeviceLicense
  • IDeviceName

The IAudioMixer capability is required for audio mixer drivers, as this tells the consuming application that the device is an audio mixer. The IAudioChannel capability must be implemented for use by the IAudioMixer implementation. All other capabilities are optional and are used to further define the device to the consuming application.

Implementation Diagram

The IAudioMixer capability defines the individual audio inputs and outputs on an audio mixer. These inputs and outputs are represented by the interface IAudioChannel.

NOTE: IAudioChannel is not a capability. It derives from the IDeviceWithCapabilities interface, meaning that each audio channel can implement its own capabilities such as IAudioChannelGainController or IAudioSignalDetection.

When a capability is implemented by the main audio mixer driver class, the features of that capability apply to the entire device. However, when a capability is implemented by a single audio channel, then it only applies to that audio channel.

NOTE: Certain consuming applications may not utilize all the capabilities supported by an audio mixer at the time of creating your driver. Regardless, all supported capabilities should still be implemented by the driver since the consuming application may be updated in the future to support the capability. This allows the driver to be feature ready without requiring an update when a new version of the consuming application is released.

Capabilities

Each audio mixer capability is described in the sections that follow. For more detailed information about each capability, refer to the C# API documentation for audio mixers.

IAudioMixer

The IAudioMixer capability is the base capability for audio mixers and must be implemented in all drivers. This capability defines a device that contains one or more audio channels.

The following methods are provided by IAudioMixer.

Method Description and Use
GetInputChannels() Gets a list of audio input channels used by this interface and by IAudioChannelGainController. Refer to the IAudioChannelGainController section below for more information.

The driver triggers an "InputChannelsChanged" event to applications when there is a change to the input channels.
GetOutputChannels() Gets a list of audio output channels used by this interface and by IAudioChannelGainController. Refer to the IAudioChannelGainController section below for more information.

The driver triggers an "OutputChannelsChanged" event to applications when there is a change to the output channels.
SetCrosspointGain(string outputId, string inputId, double gain) Sets the gain (dB) of the specified input on the specified output.

The driver triggers a "CrosspointGainChanged" event to applications when the gain level changes as described above.
GetCrosspoints() Gets all available crosspoints and their current feedback.

IAudioChannel

IAudioChannel is not a capability itself, but this interface represents an audio channel (input or output) that belongs to an IAudioMixer capability and that defines its own capabilities depending on its supported features (such as IAudioChannelGainController or IAudioSignalDetection). An audio channel may be a single connector on a device, or it may be a network input or output.

Refer to the following code sample.

Copy 
public interface IAudioChannel : IDeviceWithCapabilities
{
    /// <summary>
    /// A unique identifier for the channel. This is unique only within the containing driver instance.
    /// </summary>
    string Id { get; }

    /// <summary>
    /// The name of the channel.
    /// </summary>
    string Name { get; }

    /// <summary>
    /// Raised when there is a change to an audio channel's name (<see cref="Name"/>) property.
    /// </summary>
    event EventHandler NameChanged;

    /// <summary>
    /// Details about the physical/virtual connector on this channel.
    /// </summary>
    MediaConnector ConnectorDetails { get; }
}

IAudioChannelGainController

The IAudioChannelGainController capability is available to IAudioChannel to expose control and feedback of the gain value of the channel (in dB). The gain value for IAudioChannelGainController can be set using a percentage value (0–100), a SIMPL value (0–65535), or by entering the exact dB value. This capability also exposes mute controls on the audio channel.

The following methods are provided by IAudioChannelGainController.

Method Description and Use
SetMute(bool on) Mutes the specific channel if supported by the device. If the device supports the set mute feature, SupportsSetMute will be true; otherwise, it will be false. The Mute property is set to true if the channel is muted.

The driver triggers a "MuteChanged" event to applications when there is a change to the mute state (if SupportsMuteFeedback property is set to true).
SetDbLevel(double level) Sets a channel gain (dB) level to a specific value. The limits of this value and the step size are defined by DbLevelInfo.

DbLevel provides the current dB level with limits defined by DbLevelInfo.

The driver triggers a "LevelChanged" event to applications when there is a change to the current dB level (and if SupportsLevelFeedback property is set to true).
SetPercentageLevel(ushort percentage) Sets a channel gain (dB) level to a specified percent value (0–100).

PercentLevel provides the current dB level as a percentage (from 0 to 100%).

The driver triggers a "LevelChanged" event to applications when there is a change to the current dB level (and if SupportsLevelFeedback property is set to true).
SetSimplLevel(ushort level) Sets a channel gain (dB) level to a specified SIMPL level value (0–65535).

SimplLevel provides the current dB level as a SIMPL level (from 0 to 65535).

The driver triggers a "LevelChanged" event to applications when there is a change to the current dB level (and if SupportsLevelFeedback property is set to true).
IncreaseLevel(bool hold) Increases the dB level by one step as defined by DbLevelInfo.

The driver triggers a "LevelChanged" event to applications when there is a change to the current dB level (and if SupportsLevelFeedback property is set to true).
DecreaseLevel(bool hold) Decreases the dB level by one step as defined by DbLevelInfo.

The driver triggers a "LevelChanged" event to applications when there is a change to the current dB level (and if SupportsLevelFeedback property is set to true).

IAudioSignalDetection

The IAudioSignalDetection capability is available to IAudioChannel to expose information about the current signal that is present on the channel. This capability exposes states such as a signal being detected, a signal clipping, or whether the signal is a line‑level signal.

Refer to the following code sample.

Copy 
public interface IAudioSignalDetection : IDeviceCapability
{
    /// <summary>
    /// Returns true if the signal is clipping.
    /// </summary>
    bool SignalClipping { get; }

    /// <summary>
    /// Returns true if a signal is present.
    /// </summary>
    bool SignalPresent { get; }

    /// <summary>
    /// Returns true if the signal is a line level.
    /// </summary>
    bool IsLineLevel { get; }

    /// <summary>
    /// Returns true if the signal supports feedback reported by <see cref="SignalPresent"/>.
    /// </summary>
    bool SupportsSignalPresetFeedback { get; }

    /// <summary>
    /// Returns true if the signal supports feedback reported by <see cref="SignalClipping"/>.
    /// </summary>
    bool SupportsSignalClippingFeedback { get; }

    /// <summary>
    /// Returns true if the signal supports feedback reported by <see cref="IsLineLevel"/>.
    /// </summary>
    bool SupportsIsLineLevelFeedback { get; }

    /// <summary>
    /// Raised when there is a change to the specified channel's <see cref="SignalClipping"/> property.
    /// </summary>
    event EventHandler SignalClippingChanged;

    /// <summary>
    /// Raised when there is a change to the specified channel's <see cref="SignalPresent"/> property.
    /// </summary>
    event EventHandler SignalPresentChanged;

    /// <summary>
    /// Raised when there is a change to the specified channel's <see cref="IsLineLevel"/> property.
    /// </summary>
    event EventHandler IsLineLevelChanged;
}

IDeviceLicense

The IDeviceLicense capability is available as a base capability for audio mixer devices. This capability exposes information about any licenses configured on the device.

IChildDeviceContainer

The IChildDeviceContainer capability is available as a base capability for audio mixer devices. This capability supports reporting any child devices, such as microphones or speakers. Child devices are IDeviceWithCapabilities instances that may implement the following capabilities:

  • IBasicInformation
  • IConnection3
  • IFeedbackInformation
  • IDeviceDiagnostics
  • IDeviceIdentification
  • IDeviceName
  • IRebootController

IDeviceDiagnostics

The IDeviceDiagnostics capability is available as a base capability for audio mixer devices or as a capability on a child device. This capability exposes general diagnostic information about a device, such as its firmware version or uptime.

IDeviceIdentification

The IDeviceIdentification capability is available as a base capability for audio mixer devices or as a capability on a child device. This capability exposes control and feedback for identifying a physical device. For example, if a device supports flashing an LED for physical identification, this capability would expose controls to set that state.

IDeviceName

The IDeviceName capability is available as a base capability for audio mixer devices or as a capability on a child device. This capability exposes control and feedback for the current name assigned to the device.

Related Capabilities

The following capabilities are not specific to audio mixers but may be useful when creating the driver. For more detailed information, refer to the C# API documentation.

  • IRebootController
