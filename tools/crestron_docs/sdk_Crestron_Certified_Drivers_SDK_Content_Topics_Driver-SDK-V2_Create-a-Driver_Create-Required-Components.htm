Driver SDK V2 (Entity Model) / Create a Driver / Create Required Components

Create Required Components

The following procedure describes how to create components required for all drivers.

First, create a class to represent the root entity in your driver. Properties and commands will be added to this class in a later topic.

Copy 
using Crestron.DeviceDrivers.EntityModel;
using Crestron.DeviceDrivers.SDK;
using Crestron.DeviceDrivers.SDK.EntityModel;

public class MyDriverEntity : ReflectedAttributeDriverEntity
{
    public MyDriverEntity(DriverControllerCreationArgs args, DriverImplementationResources resources) : base(DriverController.RootControllerId)
    {
        // We'll use args and resources later.
        // The args are provided by the application.
        // The resources are loaded by the Driver SDK from the Driver JSON file
    }
}

Next, create a JSON file with the same name as the driver assembly but with a .json extension. The JSON file must be added to the project as an embedded resource.

The information below can be used to populate the JSON file. Change the default generic values to match your device, company, and so forth.

Copy 
{
    "SchemaVersion": "2.0",
    "GeneralInformation": {
        "MinSdkVersion": "21.0000.0000",
        "DriverVersion": "1.0000.0000",
        "DeviceType": "Lights",
        "Manufacturer": "SomeCompany",
        "BaseModel": "SomeDevice",
        "OtherSupportedModels": [
        ],
        "SupportedSeries": [
            "SomeSeries"
        ],
        "DependencyGroup": "",
        "Developer": {
            "Company": "My Company",
            "Contact": "My Name",
            "Email": "my@companyemail.com",
            "Website": "www.mycompanywebsite.com",
            "PhoneNumber": "(555) 555-5555"
        }
    },
    "ConfigurationSteps": {}
}

Finally, create the entry point class, which must have a default (parameterless) constructor, derive from DriverAssemblyEntryPoint, and be referenced by an assembly‑level DriverAssemblyEntryPointAttribute.

When the application loads the driver assembly, it will create an instance of this class and call CreateDriverControllerInstance to create your driver.

Copy 
using Crestron.DeviceDrivers.EntityModel;
using Crestron.DeviceDrivers.SDK;
using Crestron.DeviceDrivers.SDK.EntityModel;

// This attribute tells applications loading this assembly where to start to initialize the driver
[assembly: DriverAssemblyEntryPoint(typeof(EntryPoint))]

public sealed class EntryPoint : DriverAssemblyEntryPoint
{
    public override DriverController CreateDriverControllerInstance(DriverControllerCreationArgs args)
    {
        // Load up resources from the driver json
        var resources = DriverImplementationResources.FromCreationArgs(args, typeof(EntryPoint));

        // Create driver instance
        var driverEntity = new MyDriverEntity(args, resources);

        // Create the data structure which holds the entity and its configuration controller
        // to pass to DispatchingDeviceController below
        // TODO: Replace null here with configuration controller for the driver
        // (done in later part of tutorial)
        var entity = new ConfigurableDriverEntity(driverEntity.ControllerId, driverEntity, null);

        // This class implements DriverController for us, handling any routing of APIs to any sub-devices
        // or sub-entities we may have.
        return new DispatchingDeviceController(entity, args, null);
    }
}

Completing this process yield complete driver that has no configuration and supports nothing. These items will be added in later topics.
