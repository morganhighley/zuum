Entry Device Test Plans

The following test plans will be used to test drivers for entry (garage door and gate) devices.

Generic SIMPL Test Plan

Create a SIMPL program, and then define the wrapper module for the device type you are testing. Test all inputs and outputs of the SIMPL wrapper module and verify that they work with your driver. For more information, refer to Load a Driver.

Crestron Home Verification Points

The following testing items are specific to driver behavior within Crestron Home.

Configuration
  1. Load the driver into the import folder on the CP4-R/MC4-R, then navigate to Third Party Devices in the Pair Devices page and wait for the list to update. Verify the manufacturer shows under the device type, and all supported models from the driver show under the manufacturer without any typos.
  2. Add the driver to a room, and verify the add device dialog appears with the appropriate connection fields (IP/Hostname field).
  3. Verify any additional attributes on the driver show in subsequent dialogs with the appropriate descriptions and fields.
  4. Verify the driver is successfully added to the room.

Settings
  1. Select Settings. Verify that About, Control, and Advanced menu options are available.
  2. Select About. Verify that the device name, room name, type, manufacturer, model, version, and control type are displayed. Verify that device status is displayed and is reporting as online when the device is connected and offline when the device is disconnected.
  3. Select Control. Verify that the IP address and IP port are displayed.
  4. Select Advanced. Verify that an Enable Logging check box is displayed. Verify that the driver logs to the Crestron console when the check box is selected and stops logging when deselected.

Entry Functions
  1. In the Access section of the run time app, select Open All. Verify that the state changes to OPENING... and then OPEN once the action has completed.
  2. Select Open All several times quickly. Verify that the feedback occurs on the first button selection. Verify that the feedback is not extended or does not toggle.
  3. Select Close All. Verify that the state changes to CLOSING... and then CLOSED once the action has completed.
  4. Select Close All several times quickly. Verify that the feedback occurs on the first button selection. Verify that the feedback is not extended or does not toggle.
  5. In the Controls section, turn the Garage Door toggle on or off from its current state. Verify that the state changes to the opposite of the previous state once the action has completed. For example, turning the toggle on should change the state from CLOSED to OPEN.
  6. Turn the Garage Door toggle on or off again to return it to its original state. Verify that the state changes back to the original state the action has completed.
  7. Place the driver in a jammed state (if possible). Verify that the driver reports a JAMMED error.
  8. Place the driver in an object in path state (if possible). Verify that the driver reports an OBJECT IN PATH error.
  9. Place the driver in a manually stopped state (if possible). Verify that the driver reports a MANUALLY STOPPED error.
  10. Clear all errors and return the driver to a normal operating state. Verify that the errors are shown as cleared and that the driver is operating normally.

System Tests
  1. Simulate a power outage in the house by removing power from the processor, device, and network for at least 120 seconds before restoring power. Verify once the program loads, the driver reconnects to the device and all functionality is restored within 60 seconds.
  2. Simulate a network outage by disconnecting all network cables from the device and processor and waiting at least 120 seconds before restoring. Verify the device and driver reconnect and all functionality is restored within 60 seconds.
  3. Leave the driver connected to the device for a minimum of 24 hours, periodically checking functionality. After 24 hours, the driver should still be connected, all functionality should still work, and there should be no degradation in performance (functions should continue to work immediately and feedback should remain prompt and accurate).
  4. Delete the driver from the system. The driver should no longer be listed in the room. No exceptions should be thrown in the error log.
