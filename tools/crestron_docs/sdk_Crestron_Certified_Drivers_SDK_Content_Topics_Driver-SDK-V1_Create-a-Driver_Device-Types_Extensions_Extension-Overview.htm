Overview

A Crestron Home Extension driver is a driver for Crestron Home® OS that has a custom user interface. Crestron Home Extension drivers can be any device type since the driver developer decides how the end user interacts with the device via their custom user interface. The custom user interface is defined by the UI definition, which is an XML file containing the layout of the UI.

A Crestron Home Extension driver is in the form of a .pkg file like all other drivers, but the package contains additional components that make up the Crestron Home Extension driver:

  • A .dll file
  • A .dat file that is a JSON formatted file containing driver data
  • A UiDefinitions folder containing the UI definition file for the Crestron Home Extension driver
  • A Translations folder containing the translation files for the Crestron Home Extension driver

Crestron Home Extension Device Components

A Crestron Home Extension device is made up of the following components:

  • UI Definition
  • Language Translations
  • Property Definitions
  • Property Values

Using the UI definition, translations, and properties, Crestron Home can send user actions to your device driver via controls such as buttons and text entry boxes. These actions are called on your driver via the methods DoCommand and SetDriverPropertyValue. For example, if a user presses a button, Crestron Home will make a call to DoCommand and pass in the command string and parameters that were defined in your UI definition.

Each component that makes up a Crestron Home Extension device is described in the sections that follow.

UI Definition

The purpose of the UI definition is to define the layout of the UI that the end user will interact with through the Crestron Home app. The UI definition also defines the commands that are sent from the UI to the driver when certain actions are performed, such as a button press. The UI definition can use keys from the language translations to display static text or it can use the property values to display information from the device itself. For example, you can have a property named “PowerState” that has the value “On” or “Off,” which can then be displayed in the UI.

For more information on creating a UI definition file, refer to UI Definition.

Language Translations

Language translations provide a way for your driver to be used in any language that the developer decides to support. They are written as key/value pairs, where the keys are referenced by the UI definition and the values are the translated strings.

For more information on creating language translations, refer to Translation Files.

Property Definitions and Values

Within the driver code, you can define properties that are used to display information on the UI. A property definition is used to define the property, whereas a property value represents a specific instance of a property. Properties can be any primitive type such as a string, int, or bool. They can also be a more complex type such as an object or an object list. Objects can contain multiple primitive type properties or even other nested objects, and object lists can contain multiple instances of an object as long as all objects in the list have the same subtype (the type of the object, similar to the name of a class in object-oriented programming).

Refer to the following example that demonstrates data for each type of property:

  • Primitive Property (string, int, bool):
    • Name (string)
    • Age (int)
  • Object Property:
    • Student (object) – an object with 2 properties (Name and Age)
      • Name (string)
      • Age (int)
  • List Property:
    • School (object list) – a list of students that attend the school
      • Student (object)

Crestron Home Extension Device Framework

An Crestron Home Extension drier can be created by implementing the abstract class AExtensionDevice, which has much of its functionality already implemented. Below is an overview of all the methods on AExtensionDevice and a description on how to use and implement them.

AExtensionDevice

The following methods are provided for the AExtensonDevice class.

Abstract Methods

These methods are part of the abstract class AExtensionDevice. The driver developer must implement these methods since they will need to communicate with the actual device.

Method Description and Use
DoCommand(string command, string[] parameters) This method is called by Crestron Home. It is used to send a command to the device when a user interacts with a control on the UI that has a command action assigned to it.

The command and parameters sent are defined in the UI definition.
SetDriverPropertyValue<T>(string propertyKey, T value) This method is called by Crestron Home. It is used to set the value of a property on the device. In the UI definition, controls with output bindings will call this method whenever the value changes.

The propertyKey is defined in the UI definition and the value is provided by the user.
SetDriverPropertyValue<T>(string objectId, string propertyKey, T value) This method is called by Crestron Home. It is used to set the value of a property that belongs to an object. The objectId is a unique ID that is generated when you call CreateObject on AExtensionDevice.

The propertyKey is defined by the Properties on the DeviceClassDefinition.

Virtual Methods

These methods are part of the abstract class AExtensionDevice. The default implementation of these methods will work in most cases such as when there is only one UI definition file provided in the driver .pkg file. However, when custom logic is needed, these methods can be overridden.

Method Description and Use
GetUiDefinition(string uiFolderPath) This method is called by Crestron Home to get the UI definition for your driver. If your driver only has one UI definition (which should be the case for most drivers), then this method does not need to be overridden.

If your driver has more then one UI definition, then this method must be overridden. The uiFolderPath is the path to the folder containing the UI definition file(s) that you provided in your driver’s PKG file. If you would like to return the contents of one of the UI definition file(s) from the PKG file, use Crestron.SimplSharp.CrestronIO to read the file. If you do not want to provide the UI definition with the driver’s PKG file, you can alternatively use this method to retrieve the UI definition from anywhere, such as a variable within the driver’s code or even a web API.
GetLanguageTranslations(string translationsFolderPath, string culture) This method is called by Crestron Home to get all of the languages that this driver supports. Each language is known as a culture, and they are in the format of languagecode2-country/regioncode2 as defined by the ISO 639-1 standard and the ISO 3166 standard (Ex. “en-US” for English in the United States).

If the names of the language translation files provided in your driver’s PKG file are in this format, then this method does not need to be overridden.
SetDriverPropertyValue<T>(string objectId, string propertyKey, T value) This method is called by Crestron Home to get the language translations for the specified culture.

If the names of the language translation files provided in your driver’s PKG file are in the correct format (languagecode2-country/regioncode2 as defined by the ISO 639-1 standard and the ISO 3166 standard), then this method does not need to be overridden.

Protected Methods

These methods are part of AExtensionDevice and are used to create and modify the values of properties as well as raise events when needed. Properties are used to display information to the user or to receive input from them. Property values that are changed by the user will be sent to the driver via one of the SetDriverPropertyValue methods. These methods below are used to create and modify the value of a property based on feedback from the device.

Method Description and Use
CreateClassDefinition(string className) This method is used to define a class, much like defining a class in object-oriented programming. This method returns an empty ClassDefinition that can have properties added to it via ClassDefinition.AddProperty. Only one definition for a class can exist and, therefore, the className provided must be unique. However, multiple instances of the class can exist.

A class can be instantiated using the method CreateObject, which returns an ObjectValue.
CreateProperty<T>(PropertyDefinition propertyDefinition) This method is used to create a property. The value of this property can be bound to a control in the UI definition to display data to the user or receive input from the user (Ex., A property can be set as the line1label of a textdisplay control to display information to the user, or it can be set to the value of a textentry control to receive input from the user).

In the case of a textentry control, if the user modifies the property’s value, the method SetDriverPropertyValue will be called.
CreateObject(ClassDefinition classDefinition) This method is used to create an instance of a class that has been previously defined.

This method will return an ObjectValue that contains all the properties defined by the ClassDefinition.
CreateList(PropertyDefinition listDefinition) This method is used to create a list of objects and will return a ObjectList.
DeleteProperty(PropertyValue propertyInstance) This method is used to delete the instance of a property, object, or list. If the object being deleted is a member of a list it will be removed from the list.
Commit() This method is used to raise an event to the UI with all the changes to property values since the last time Commit was called so that they are updated on the UI. This allows you to change the value of many properties at once and then call Commit to send the event to the UI.

Commit can be called whenever necessary and should always be called within a reasonable time after changing to value of a property so that the UI is displaying the most up-to-date value.
RaiseUiDefinitionChangedEvent() This method is used to tell Crestron Home that the UI changed. This method only needs to be used if GetUiDefinition is overridden and only if the UI definition changes dynamically based on the device.

In most cases, the UI should be static with only properties that change and, therefore, this method is not needed.
RaiseSupportedCulturesChangedEvent() This method is used to tell Crestron Home that the supported cultures changed.

This method only needs to be used if GetSupportedCultures is overridden.
RaiseLanguageTranslationsChangedEvent() This method is used to tell Crestron Home that the language translations changed. This method only needs to be used if GetLanguageTranslations is overridden.

PropertyDefinition

PropertyDefinition is the implementation of IPropertyDefinition. A PropertyDefinition is used to define a property that can be linked to a control in the UI. A property’s values can be any of the primitive types such as string, int, and bool, or it can be an object or an object list. The class PropertyDefinition has four constructors, each used to create a different type of property. The use of each contructor is explained in more detail below:

  • public PropertyDefinition(string key, string nameLocalizationKey, DevicePropertyType type)

    • This constructor is used to create a simple property definition that has a primitive type such as a string, int, or bool.

  • public PropertyDefinition(string key, string nameLocalizationKey, DevicePropertyType type, IClassDefinition classDefinition)

    • This constructor is used to create a property that is of type DevicePropertyType.Object or DevicePropertyType.ObjectList. For an object, the classDefinition is the class that the object is. For object list, the classDefinition is the class that the list contains.

  • public PropertyDefinition(string key, string nameLocalizationKey, DevicePropertyType type, IEnumerable<IPropertyAvailableValue> availableValues)

    • This constructor is used to create a property that has a set amount of available values. The available values provided are the only values that this property can be set to.

  • public PropertyDefinition(string key, string nameLocalizationKey, DevicePropertyType type, double minValue, double maxValue, double stepSize)

    • This constructor is used to create a property that has a min, max, and step size. The minValue is the lowest value that the property can be set to and the maxValue is the highest value that the property can be set to. The stepSize is the how much the value must be incremented by. For example, for a min of 0, max of 6, and step size of 2, the values that the property can be set to are 0, 2, 4, and 6.

The following members are provided for this class:

Member Description and Use
Key A unique identifier for this property. The key must be unique across all properties in the driver.
NameLocalizationKey A key used to look up the name of the property in the translation dictionary.
Type The property type.
Subtype This is only used for properties that are of Type DevicePropertyType.Object or DevicePropertyType.ObjectList.

For objects, this is the type that the object is as defined in the DeviceClassDefinition. For object lists, this is the type of object that the list contains.
AssociatedClass If this property belongs to an object, then this is the type of the object that this property belongs to as defined in the object's ClassDefinition.
AvailableValues This is the list of values that this property can be set to.
MinValue The minimum value that this property can be set to. For numeric types, this is the minimum value. For strings, this is the minimum length of the string.
MaxValue The maximum value that this property can be set to. For numeric types, this is the maximum value. For strings, this is the maximum length of the string.
StepSize The increment size that this property can be set to starting at minimum and ending at the maximum. This is only valid for numeric type properties.

ClassDefinition

ClassDefinition is the implementation of IClassDefinition. A ClassDefinition is used to define an object property. This class does not have a public constructor and can be created only by calling the method CreateClassDefinition.

The following members are provided for this class:

Member Description and Use
ClassName The name of the class, which is similar to the name of a class in object-oriented programming.
Properties The properties that this object contains.
AddProperty This method is used to add properties to the class. When a class is initially created using CreateClassDefinition there are no properties on the object.

PropertyAvailableValue

PropertyAvailableValue is the implementation of IPropertyAvailableValue. A PropertyAvailableValue is used to define a specific value that is available to a property.

The following members are provided for this class:

Member Description and Use
Label This is the label of the available value that will appear in the UI. However, if the LabelLocalizationKey is provided, then the translated value will be prioritized over the label.

To use the label instead of a translation, set LabelLocalizationKey to an empty string or null. This should only be used when the value you want to display can’t be translated; otherwise, a translation key should be used instead.
LabelLocalizationKey A key to look up the user‑visable translation for this value in the translation dictionary.
Enabled If this is set to true, then this value will be selectable in the UI. If this is false, then this value will not be selectable.

PropertyAvailableValue<T>

The following member is provided for this class:

Member Description and Use
Value The internal value that the driver will receive. This does not appear in the UI.

PropertyValue

PropertyValue is the implementation of IPropertyValue. A PropertyValue is the instance of the property that contains the current state. A PropertyValue is further derived depending on the device type. For all primitive types (string, int, bool, and so forth) the property value is PropertyValue<T>, for objects it is ObjectValue, and for lists it is ObjectList.

The following members are provided for this class:

Member Description and Use
Id This is a GUID to uniquely identify every property within the driver. The Id of the property is generated when the property is created. This property is read-only.
DefinitionKey This key links this property value to the definition of the property. This property is read-only.
Type This represents the type of the property as defined by in the property’s definition.
Enabled This is a flag to determine if this property is enabled or disabled. Depending on the UI that this property is displayed on, if the property is disabled, it will not selectable or editable by the user.
IsValueAvailable This is a flag to determine if the property value is currently available. An example use case would be a thermostat where the value of the "Temperature" property is 70, but the thermostat is offline so the actual value is not currently available.

PropertyValue<T>

PropertyValue<T> is the implementation of IPropertyValue<T>. To create a new PropertyValue<T>, call CreateProperty<T> and pass in the definition of the property.

The following member is provided for this class:

Member Description and Use
Value This is the current value of the property. You can set the value of this property to update it in the UI.

For property value changes to be sent to the UI, Commit() must be called.

ObjectValue

ObjectValue is the implementation of IPropertyObjectValue. To create a new ObjectValue instance, call CreateObject and pass in the definition of the class.

The following member is provided for this class:

Member Description and Use
SubType The name of the class defining this object.
GetValue<T>(string propertyKey) Used to get the value of a property on this object. The value returned can be modified to update it to a new value.
GetObjectValue(string propertyKey) Used to get the value of a property this is another object (nested object).
GetListValue(string propertyKey) Used to get the value of property this is a list. The list returned can be modified as needed. 
SetObjectValue(string propertyKey, Object value) Sets the value of a property that is an object (nested object). When this object is created, all nested objects will initially be empty.
SetListValue(string propertyKey, ObjectList value) Sets the value of a property that is a list. When this object is created, all list properties will initially be empty.

ObjectList

ObjectList is the implementation of IPropertyListValue. To create a new ObjectList instance, call CreateList and pass in the definition of the property. The Type of the property definition must be DevicePropertyType.ObjectList.

The following member is provided for this class:

Member Description and Use
SubType The name of the class defining the objects that this list contains. An object list cannot contain objects from two different classes.
AddObject(Object objectToAdd) Adds an object instance to the list.
RemoveObject(string objectId) Removes an object instance from the list.

This does not delete the object. However, deleting the object will remove it from the list.
Clear() Remove all objects from the list.

This does not delete the objects.

OperationResult

OperationResult is the implementation of IOperationResult. An OperationResult is the returned value for DoCommand and SetDriverPropertyValue. If sending the command or setting the property is a success, then the result should be returned as a success. If the operation fails, then the result should be returned as an error with an optional message to be displayed to the user.

The following member is provided for this class:

Member Description and Use
ResultCode The result of the action, which is either Success or Error.
UserMessage If the result code is Error, this message will be displayed to the user.
