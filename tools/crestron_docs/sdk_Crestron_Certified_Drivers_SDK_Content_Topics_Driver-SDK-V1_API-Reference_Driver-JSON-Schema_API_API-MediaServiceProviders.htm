MediaServiceProviders

The MediaServiceProviders property specifies all controllable and uncontrollable media service applications, such as Netflix®, Hulu®, and Spotify® services. This property also specifies the component interfaces and their associated Support<feature> properties that are supported on each media service.

MediaServiceProviders Properties

Property Use
SupportsMediaServices (bool)

Specifies if media services are supported.
HasDirectAccessToApps (bool) Specifies if this driver can issue commands to select media services.
SupportsActiveMediaServiceFeedback (bool) Specifies if the driver supports reporting the current active media service.
SupportsActiveMediaServiceStateFeedback (bool) Specifies if the driver supports reporting the state of the current active media service.
SupportsMediaServiceSubscriptionStateFeedback (bool) Specifies if the driver supports reporting the subscription state of the current active media service.
Available Services (<ListObject>) A list of available media services as defined in the following table.
MediaServicePollCommand (string) The command sent to the device that polls it for the active media service. This does not have to be set if the InputPoll command retrieves this information.
MediaServicePollingEnabled (bool) Specifies if the driver should send the MediaServicePollCommand. Set this to false if InputPoll is retrieving the media service feedback.
MediaServiceSubscriptionStatePollCommand (string) The command sent to the device to get the current subscription state of a media service.
MediaServiceSubscriptionStatePollingEnabled (bool) Set to true if MediaServiceSubscriptionStatePollCommand should be sent to the device
MediaServicePlaybackStatePollCommand (string) The command sent to the device to get the current playback state of the active media service.
MediaServicePlaybackStatePollingEnabled (bool) Set to true if MediaServicePlaybackStatePollCommand should be sent to the device.

A media service is defined with the following properties.

MediaServiceProviders Properties - Definitions

Property Use
Id

A unique ID for the media service.
FriendlyName A user-friendly name for the media service.
Command The command string to send to the device to select this service.
IsBranded Set to true if FriendlyName is a branded name.
IsSelectable Set to true if the media service is selectable. If a media service is not selectable, then it cannot be supported by ActiveMediaServiceFeedback.
SupportedFeatureData Defines the component interfaces that are supported when this media service is in use and the Supports<feature> properties are still supported.

Media service feedback is handled with the following properties.

NOTE: Only active media service feedback is supported in the driver framework. A driver can implement this on its own.

MediaServiceProviders Properties - Feedback

Property Use
GroupHeader

A unique ID for the media service.
ActiveServiceFeedbackData Contains a group header for messages specifically about the active media service, if applicable. Feedback is a Dictionary<string, string>, where the key is the media service ID defined in AvailableServices, and the value is the response from the device. Drivers can override DeConstructActiveMediaServiceFeedback and modify the response to match the values specified here.
PlaybackStateFeedbackData Contains a group header (which can be null) for messages specifically about the play state feedback for the active media service. This is supported only if the manufacturer provides this functionality for the media service.

  • The key is the status (Playing, Paused, Stopped, NotActive).
  • The value is the response from the device for the service.
  • The reported status is assumed to be for the active service since more than one service cannot be active at the same time.
