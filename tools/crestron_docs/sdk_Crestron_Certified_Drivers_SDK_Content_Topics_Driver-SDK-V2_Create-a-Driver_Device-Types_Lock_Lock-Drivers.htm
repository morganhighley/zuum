Lock Drivers

Any entity model driver that supports locking and unlocking may implement the lock capability. The lock capability exposes controls for some common lock settings as well as lock event feedback. If the device also supports user or pin management, the entryUserManagement capability can be implemented by the driver.

Lock drivers may implement additional capabilities. For example, a driver that controls a lock and reports whether an entryway is open or closed could implement both the lock and entry capabilities. If the driver will control multiple lock devices, then it should be configured as a Platform Drivers device that exposes individual lock devices as managed devices.

To locate a self-test plan for lock device drivers, refer to Lock Test Plans.
