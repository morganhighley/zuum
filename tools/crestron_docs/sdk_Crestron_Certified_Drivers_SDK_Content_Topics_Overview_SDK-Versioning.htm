Overview / SDK Framework Versioning

SDK Framework Versioning

As Crestron incorporates new interfaces, the major version of the framework will continue to advance. To ensure that the platform that you are using has the required framework interface, update to the minimum framework version required in your solution.

NOTE: To determine hardware and software versions that are supported by the various Crestron SDK frameworks, refer to SDK Framework Comparison Matrices.

The primary framework starts at version 2.x and contains support for all native device types that were not added in subsequent releases. The following list summarizes the updates provided in subsequent framework releases.

NOTE: For a complete list of features and updates for each SDK version, refer to the release notes that are provided within the SDK version download pages on the Crestron website. Refer to Resources for links to all available download pages.
  • Version 2.x: Baseline framework of Crestron Drivers functionality.
  • Version 3.x: Contains the framework that supports Crestron Home Extension devices.
  • Version 4.x: Contains the framework that supports AV switchers, security systems, and sequences for Crestron Home Extension devices.
  • Version 5.x: Contains the framework that supports event triggers for Crestron Home Extension devices. Changed the name of the "Audio Video Switcher" device type to "AV Switcher."
  • Version 6.x: Added ITcp2 interface to enable the use of a host name in drivers with platforms that support host names. Updated sample projects and added a serial security system driver example. Made bug fixes and improvements to manifest utility.
    • NOTE: Projects that reference the new RADCommon.dll must also now add a reference to Crestron.DeviceDrivers.API.dll, as it is a dependency of the updated RADCommon.dll.
  • Version 7.x: Added advanced logging interface for use with Crestron Home.
  • Version 8.x: Updated Crestron Drivers framework version to new format, fixed issue where programmable actions were not supported in all devices in a platform driver.
  • Version 9.x: Updated sample platform driver.
  • Version 10.x: Updates to IR displays and security systems. Bug fixes and performance updates.
  • Version 11.x: Bug fixes and performance updates.
  • Version 12.x: Bug fixes and performance updates.
  • Version 13.x: Added support for security system zones.
  • Version 14.x: Updated AV switcher device types. Bug fixes and performance updates.
  • Version 15.x: Bug fixes and performance updates.
  • Version 16.x: Added enhancements for security system zone support.
  • Version 17.x: Added Audio Mixer and Power Controller native device types.
  • Version 18.x: Added Network Switch device type.
  • Version 19.x: Added TSR-310 voice control support.
  • Version 20.x: ManifestUtility updated to support new developerContact object.
  • Version 21.x: Added support for new Entity Model (V2) architecture for creating drivers.
  • Version 22.x: Added support for properties via Crestron Home programming and native Entry devices.
  • Version 23.x: Added Microphone and Camera device types.
  • Version 24.x: Implemented Garage Door and Pool modules.
  • Version 25.x: Added Lock device type.
  • Version 26.x: Bug fixes and performance updates.

The SdkVersion property in the driver JSON file must be set to the minimum SDK version required for your driver to operate properly.
