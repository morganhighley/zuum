Feedback

The Feedback property is different for each device type. The base implementation of ResponseValidation will handle feedback according to the values entered for this property. All Deconstruct<Feature> methods will also refer to the driver JSON data to determine the type of feedback received and the new device state.

Display Device Feedback

Display devices support the following feedback:

  • Header
    A header that is present in all standard feedback.
  • CommandAckNak
    Two string properties for ACK and NAK.
  • PowerFeedback, EnergyStarFeedback, MuteFeedback, VideoMuteFeedback, and OnScreenDisplayFeedback
    Contains GroupHeader (string) that stores the header for this response and a dictionary where the valid keys are two states (On and Off) and the values are the device responses that correspond to those states.
  • InputFeedback
    Contains GroupHeader (string) that stores the header for this response and a list where the values are of type InputFeedbackValues that are defined as follows:

    InputFeedbackProperties

    • Property Use
    InputType (int)

    The input type that corresponds to the VideoConnections enumerable.
    Feedback (string) The response from the device for the defined input.

  • VolumeFeedback and LampHourFeedback
    Contains GroupHeader (string) that stores the header for this response.
  • SupportsUnsolicitedFeedback
    A setting used to disable automatic polling and to rely on unsolicited feedback from the device. The driver will attempt automatic polling if it changes connection states.
  • ConnectedStatePollCommand
    Contains the value that should be sent to the device at an interval to test the current connection. Serial drivers will not detect a disconnected state automatically unless there is some active communication.

A/V Receiver Device Feedback

A/V receiver devices support the following feedback:

  • Header

    A header that is present in all standard feedback.
  • CommandAckNak

    Two string properties for ACK and NAK.
  • PowerFeedback, MuteFeedback, LoudnessFeedback, and ToneStateFeedback
    Contains GroupHeader (string) that stores the header for this response and a dictionary where the valid keys are two states (On and Off) and the values are the device responses that correspond to those states.
  • InputFeedback and AudioFeedback
    Contains GroupHeader (string) that stores the header for this response and a list where the values are of type InputFeedbackValues that are defined as follows:

    InputFeedbackValues Properties

    • Property Use
    InputType/AudioInputType (int)

    The input type that corresponds to the AudioConnections/VideoConnections enumerable.
    Feedback (string) The response from the device for the defined input


  • VolumeFeedback and LampHourFeedback
    Contains GroupHeader (string) that stores the header for this response and a dictionary where the valid keys are the JSON values listed below and the values are the device response that corresponds to those states.
    • Unknown
    • Movie
    • Music
    • Game
    • Direct
    • PureDirect
    • Stereo
    • Auto
    • DolbyDigital
    • DtsSurround
    • Thx
  • ToneControlFeedback
    Contains GroupHeader (string) that stores the header for this response and a dictionary where the valid keys are two states (Bass and Treble) and the values are the device responses that correspond to those states.
  • TunerFeedback
    Contains GroupHeader (string) that stores the header for this response and an object called FrequencyBandFeedback that contains its own GroupHeader (string) and a dictionary where the valid keys are two states (Am and Fm) and the values are the devices responses that correspond to those states.
  • AudioFeedback
  • VolumeFeedback
    Contains GroupHeader (string) that stores the header for this response
  • SupportsUnsolicitedFeedback
    A setting used to disable automatic polling and to rely on unsolicited feedback from the device. The driver will attempt automatic polling if it changes connection states.
  • ConnectedStatePollCommand
    Contains the value that should be sent to the device at an interval to test the current connection. Serial drivers will not detect a disconnected state automatically unless there is some active communication.
