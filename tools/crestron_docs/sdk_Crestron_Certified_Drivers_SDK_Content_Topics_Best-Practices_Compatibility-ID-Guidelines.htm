Compatibility ID Guidelines

The CompatibilityIds property is an array of strings that specify the connector compatibilities.

  • Logical connector types (such as Stream) will require this property to be filled to connect and have media properly routed.
  • The pipe ("|") character is reserved and should not be used in any compatibility ID item. Using the pipe character in a compatibility ID may result in routing errors.
  • Each compatibility ID in the set indicates that it can only connect to other connectors of the same type and with one of the same IDs.
  • Typically, the compatibility ID will fall under two categories: standard and proprietary protocols.

    • For example, a standard audio protocol is "AES67" and a standard video protocol is "RTSP".

  • Two device drivers from different manufacturers could include this compatibility ID to be compatible with each other within the hosting application/program. Refer to table below for supported compatibility IDs.

    • For example, a streaming device in a proprietary, closed ecosystem will only allow connections within its ecosystem. Therefore, each streaming device would define an output connector of type "Stream" with one or more compatibility IDs set to any arbitrary string. As long as they match across all the streaming devices in that ecosystem, the hosting application/program would recognize them as compatible.

  • It is recommended that proprietary compatibility IDs include the manufacturer name to guarantee uniqueness across different device implementations (which prevents allowing connections to devices from a different manufacturer with an identically specified compatibility ID).
  • This property is named generically to allow associations between any similar future connector types or even physical connectors (if there is any reason to associate them beyond using their type).
  • This property may be null or omitted if not applicable.

The following table lists the standardized stream compatibility IDs and the required formats for the Location property for driver compatibility.

Compatibility ID Description Location Format Example
AES67 Audio Stream Standard host:port 239.69.0.1:5004
RTSP Real Time Streaming Protocol rtsp://[username]:[password]@[host]:[port]/[path]. rtsp://10.0.1.1:8082 rtsp://admin:password@mycamera:80/stream?ch2
