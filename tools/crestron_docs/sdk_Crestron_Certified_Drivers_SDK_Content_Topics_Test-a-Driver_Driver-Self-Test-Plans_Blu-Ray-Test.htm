Blu-ray Disc Player Test Plans

The following test plans will be used to test drivers for Blu-ray Disc® Player devices.

Generic SIMPL Test Plan

Create a SIMPL program, and then define the wrapper module for the device type you are testing. Test all inputs and outputs of the SIMPL wrapper module and verify that they work with your driver. For more information, refer to Load a Driver.

Crestron Home Verification Points

The following testing items are specific to driver behavior within Crestron Home.

Configuration
  1. Before loading the driver, add a display to the system to test with the Blu‑ray Disc Player. An arbitrary IR driver can be used if you do not have a controllable display. If the Blu‑ray Disc Player supports a separate audio output, also add an AVR to the room.
  2. Load the driver into the import folder on the CP4-R/MC4-R, then navigate to Third Party Devices in the Pair Devices page and wait for the list to update. Verify the manufacturer shows under the device type, and all supported models from the driver show under the manufacturer.
  3. Add the driver to a room, and verify the add device dialog appears with the appropriate connection fields (list of IR/COM ports, list of HDMI connectors, or IP/Hostname field).
  4. Verify any additional attributes on the driver show in subsequent dialogs with the appropriate descriptions and fields.
  5. Verify the driver is successfully added to the room.
  6. Verify the driver shows all of its outputs in the source routes page and you can connect all the other devices added in step 1 to the Blu‑ray Disc Player driver.

Controls
  1. From an end user UI, select the Blu‑ray Disc Player as the source for the room. Verify the Blu‑ray Disc Player powers on and you are presented with all supported Blu‑ray Disc Player standard controls. Verify any unsupported standard controls are hidden. Verify the controls do not disappear after ten seconds and no offline message is presented for the Blu‑ray Disc Player.
  2. Press each navigation button and verify the device immediately responds to each press. Multipress each navigation button supporting multipress, quickly (at least three times per second for at least ten times) and verify no presses are dropped or latched high. Press and hold any supporting functions and verify the device responds to the press immediately, the repsonse is smooth, and it stops as soon as the press is released.
  3. Verify each media transport functions work as they do with the device's remote.
  4. Verify each keypad button works. Multipress each digit and make sure no digit is dropped nor latched resulting in additional digits.
  5. Verify all additional supported functions show in the more tab. Verify each function works as it does with the device's remote.
  6. Power off the room and verify the device powers off after 60 seconds.
  7. Before the cool down completes, route the source again and verify it turns on once the cool down completes.
  8. Trigger a function before warm up completes, and verify the function triggers on the device once warm up completes.

System Tests
  1. Simulate a power outage in the house by removing power from the processor, device, and network for at least 60 seconds, before restoring power. Verify once the program loads, the driver reconnects to the device and all functionality is restored within 60 seconds.
  2. Simulate a network outage by disconnecting all network cables from the device and processor and waiting at least 60 seconds before restoring. Verify the device and driver reconnect and all functionality is restored within 60 seconds.
  3. Leave the driver connected to the device for a minimum of 24 hours, periodically checking functionality. After 24 hours, the driver should still be connected, all functionality should still work, and there should be no degradation in performance (functions should continue to work immediately and feedback should remain prompt and accurate).
  4. Delete the driver from the system. The driver should no longer be listed in the room. The devices routed to the Blu-ray Disc Player should no longer be selectable to be routed and powered on in the end user UIs. No exceptions should be thrown in the error log.
